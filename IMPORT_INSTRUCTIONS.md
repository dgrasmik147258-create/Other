# Инструкции по импорту Kafka Monitoring Dashboard

## 🚀 Быстрый старт

### Вариант 1: Автоматическое развертывание (рекомендуется)

1. **Запустите скрипт развертывания:**
   ```bash
   ./deploy.sh
   ```

2. **Откройте Grafana:**
   - URL: http://localhost:3000
   - Логин: `admin`
   - Пароль: `admin`

3. **Дашборд будет автоматически импортирован**

### Вариант 2: Ручной импорт

1. **Запустите инфраструктуру:**
   ```bash
   docker-compose up -d
   ```

2. **Откройте Grafana:**
   - URL: http://localhost:3000
   - Логин: `admin`
   - Пароль: `admin`

3. **Импортируйте дашборд:**
   - Перейдите в **Dashboards** → **Import**
   - Нажмите **Upload JSON file**
   - Выберите файл `kafka-monitoring-dashboard.json`
   - Нажмите **Load**

4. **Настройте источник данных:**
   - Выберите **Prometheus** из выпадающего списка
   - Нажмите **Import**

## 📊 Структура дашборда

### Секция 1: Оборудование
- **График нагрузки процессора** - показывает использование CPU в реальном времени
- **Текущая нагрузка процессора** - текущее значение CPU в процентах
- **График нагрузки диска** - использование дискового пространства
- **График нагрузки ОЗУ** - использование оперативной памяти
- **Текущая нагрузка ОЗУ** - текущее значение RAM в процентах

### Секция 2: Компоненты кластера Kafka
- **Доступность кластера** - статус недореплицированных партиций
- **JVM Threads State** - состояние потоков Java Virtual Machine
- **Request Total Time (ms)** - общее время обработки запросов
- **Кол-во текущих подключений** - активные подключения к брокерам
- **Requests Per Second** - количество запросов в секунду
- **Partition Count** - общее количество партиций
- **Active Broker Count** - количество активных брокеров
- **Local Time (ms)** - время обработки на лидер-партиции
- **Request Queue Time (ms)** - время ожидания в очереди запросов
- **Failed Producer Requests** - неудачные запросы производителей
- **Under Replicated Partitions** - недореплицированные партиции

### Секция 3: Очереди
- **Размер очереди** - количество сообщений в очереди
- **Кол-во partition per topic** - количество партиций на топик
- **Динамика лага** - график изменения лага консьюмеров
- **Очереди без сообщений за последний 24 часа** - неактивные очереди
- **Кол-во consumer per topic** - количество консьюмеров на топик
- **Общее состояние лага** - цветовая индикация лага

### Секция 4: Consumer
- **Bytes Consumed Rate** - скорость потребления байт
- **Rebalance Time Max** - максимальное время ребалансировки
- **Скорость обработки сообщений** - количество сообщений в секунду
- **Доступность** - статус доступности консьюмеров
- **Динамика балансировки** - график ребалансировки групп

## 🎨 Цветовая схема

### Пороги для CPU и RAM:
- 🟢 **Зеленый**: < 70% (CPU) / < 80% (RAM) - нормальное состояние
- 🟡 **Желтый**: 70-90% (CPU) / 80-90% (RAM) - предупреждение
- 🔴 **Красный**: > 90% - критическое состояние

### Пороги для диска:
- 🟢 **Зеленый**: < 80% - нормальное состояние
- 🟡 **Желтый**: 80-90% - предупреждение
- 🔴 **Красный**: > 90% - критическое состояние

### Пороги для лага консьюмеров:
- 🟢 **Зеленый**: ≤ 50 - нормальное состояние
- 🟡 **Желтый**: > 50 < 100 - предупреждение
- 🔴 **Красный**: ≥ 100 - критическое состояние

## ⚙️ Настройка метрик

### Требуемые метрики:

1. **Node Exporter метрики:**
   - `node_cpu_seconds_total`
   - `node_memory_MemAvailable_bytes`
   - `node_filesystem_avail_bytes`

2. **Kafka метрики:**
   - `kafka_server_replicamanager_underreplicatedpartitions`
   - `kafka_network_requestmetrics_totaltimems`
   - `kafka_server_socket_server_metrics_connection_count`
   - `kafka_controller_kafkacontroller_partitioncount`
   - `kafka_controller_kafkacontroller_activebrokercount`
   - `kafka_log_logendoffset`
   - `kafka_log_logstartoffset`
   - `kafka_consumer_group_max_lag`

3. **JVM метрики:**
   - `jvm_threads_state`
   - `jvm_memory_heap_used`
   - `jvm_gc_collection_count`

### Настройка Kafka Exporter:

```yaml
# В docker-compose.yml измените адрес Kafka:
kafka-exporter:
  command:
    - '--kafka.server=YOUR_KAFKA_HOST:9092'
```

### Настройка Prometheus:

```yaml
# В prometheus-kafka-config.yml измените адреса:
scrape_configs:
  - job_name: 'kafka-brokers'
    static_configs:
      - targets: 
        - 'YOUR_KAFKA_HOST:9308'
```

## 🔧 Кастомизация

### Добавление новых метрик:

1. Откройте дашборд в режиме редактирования
2. Нажмите **Add Panel**
3. Выберите **Add a new panel**
4. Настройте PromQL запрос
5. Сохраните изменения

### Изменение порогов:

1. Выберите панель
2. Перейдите в **Field** → **Thresholds**
3. Настройте значения и цвета
4. Сохраните

### Изменение временного диапазона:

1. В правом верхнем углу выберите временной диапазон
2. Или используйте предустановленные значения:
   - Последний час
   - Последние 6 часов
   - Последние 24 часа
   - Последние 7 дней

## 🐛 Устранение неполадок

### Метрики не отображаются:

1. **Проверьте Prometheus:**
   - Откройте http://localhost:9090
   - Перейдите в **Status** → **Targets**
   - Убедитесь, что все таргеты в состоянии **UP**

2. **Проверьте Kafka Exporter:**
   - Откройте http://localhost:9308/metrics
   - Убедитесь, что метрики Kafka доступны

3. **Проверьте Node Exporter:**
   - Откройте http://localhost:9100/metrics
   - Убедитесь, что метрики системы доступны

### Неточные данные:

1. **Проверьте интервал сбора:**
   - В Prometheus: **Status** → **Configuration**
   - Убедитесь, что `scrape_interval: 15s`

2. **Проверьте PromQL запросы:**
   - В Grafana откройте панель в режиме редактирования
   - Проверьте корректность запроса в **Query**

### Проблемы с производительностью:

1. **Увеличьте интервал обновления:**
   - В настройках дашборда измените **Auto-refresh**
   - Рекомендуется: 30s или 1m

2. **Оптимизируйте запросы:**
   - Используйте recording rules из `kafka-rules.yml`
   - Уменьшите временной диапазон

## 📞 Поддержка

При возникновении проблем:

1. Проверьте логи контейнеров:
   ```bash
   docker-compose logs grafana
   docker-compose logs prometheus
   docker-compose logs kafka-exporter
   ```

2. Проверьте статус сервисов:
   ```bash
   docker-compose ps
   ```

3. Перезапустите сервисы:
   ```bash
   docker-compose restart
   ```
